# WC-L — фронтенд: устройство и изменения

Этот документ описывает **статический интерфейс** сайта (`frontend/`), решения по вёрстке и поведению, чтобы другой разработчик мог открыть код и понять, **что за что отвечает**. Корневой `README.md` в репозитории относится к бэкенду (FastAPI); здесь — только фронт.

---

## Структура каталога

| Путь | Назначение |
|------|------------|
| `index.html` | Одна страница (SPA без фреймворка): шапка, все «экраны» в `<main>`, подвал внизу. |
| `static/css/fantasy.css` | Все стили: тема, фон, слои, компоненты, футер. |
| `static/js/app.js` | Навигация между разделами, API, токен, профиль, админка, анимации монет/звёзд. |
| `static/fonts/` | Локальные шрифты (Montserrat). |
| `static/img/` | Фоновое изображение подземелья, GIF монет и звёзд и т.д. |

Сервер отдаёт `frontend/` как статику (см. `app/main.py`); пути в HTML — относительные (`static/...`).

## Как устроена страница (`index.html`)

### Одна HTML-страница, много «экранов»

- В `<main>` лежат **секции** с `id="view-<имя>"` и классом `view`.
- Одновременно видна **только одна** секция: у неё есть класс `is-visible`, у остальных его нет.
- Переключение делает **JavaScript** (`showView` в `app.js`), а не отдельные URL.

Имена разделов: `home`, `about`, `rules`, `news`, `connect`, `login`, `register`, `profile`, `admin`.

### Атрибут `data-view` на `<body>`

При каждом переключении экрана выставляется:

```html
<body data-view="home">
```

и в коде:

```js
document.body.setAttribute("data-view", name);
```

**Зачем:** раньше для CSS пробовали селектор `:has()`; он не везде предсказуем. `data-view` — явное состояние, его можно использовать в стилях или отладке (в DevTools видно текущий экран).

### Подвал и нижняя полоса

Разметка внизу страницы:

```html
<div class="site-footer-band">
  <div class="footer-gold-line" aria-hidden="true"></div>
  <footer class="site-footer">...</footer>
</div>
```

| Элемент | Роль |
|---------|------|
| `site-footer-band` | **Непрозрачная** нижняя полоса: текст и золотая линия не лежат на фоновом фото подземелья (см. раздел про фон). |
| `footer-gold-line` | Тонкая горизонтальная линия — визуальная граница между зоной с картинкой (контент выше) и серым низом. |
| `site-footer` | Текст про WC-L, статус API, ссылки на `/docs` и health. |

Подсказка про Live Server и порт 8000 — элемент `#wc-l-live-hint` с классом `footer-live-hint`: по умолчанию скрыта в CSS, показывается из JS, если API открыт с **абсолютного** origin (например Live Server → бэкенд на 8000).

### Параметр `?v=` в ссылках на CSS/JS

В `index.html`:

```html
<link rel="stylesheet" href="static/css/fantasy.css?v=7" />
<script src="static/js/app.js?v=4"></script>
```

**Зачем:** сброс кэша браузера после правок. При следующем крупном изменении стилей/скрипта номер можно увеличить.

## Стили (`static/css/fantasy.css`)

### Переменные `:root`

В начале файла заданы цвета, шрифты, границы (`--border-gold`, `--bg-deep`, `--bg-card` и т.д.). От них зависят панели, кнопки, футер.

### Фон страницы и слои (важно для чтения кода)

1. **`body`** — фоновое изображение подземелья (`stock-vector-dungeon-...jpeg`), поверх него полупрозрачный градиент, затем паттерн. Это **полноэкранный** фон документа.
2. **`body::before`** — дополнительные радиальные градиенты (атмосфера), `z-index: 0`, без взаимодействия с мышью.
3. **`main`** — контент, `z-index: 3`, чтобы быть **над** слоем монет.
4. **`.fx-orbs-layer`** — контейнер для анимированных монет (GIF), создаётся из JS, `z-index: 2` (между фоном и контентом).
5. **`body > .site-footer-band`** — нижняя полоса с **плотным** тёмным фоном и лёгкой тенью вверх, `z-index: 3`, как у `main`.

Так достигается, что **текст подвала не читается на светлых фрагментах jpeg**: полоса `site-footer-band` полностью перекрывает фон body в своей области.

### Золотая линия и отступ сверху

- **Отступ между концом `main` и началом нижней полосы** задаётся у `site-footer-band`:  
  `margin-top: calc(9.5rem - 0.28rem + 3px)` — это историческое значение под визуальный зазор и слой монет (в комментариях в CSS указано назначение).
- Линия **`.footer-gold-line`** внутри полосы — без лишних отступов, это первый пиксель «серого низа».

### Компоненты контента

- **`.view` / `.view.is-visible`** — скрытие/показ секций (`display: none` / `block`).
- **`main`** — `max-width: 1100px`, центрирование, внутренние отступы.
- **`.hero`**, **`.card-grid`**, **`.feature-card`** — главная страница.
- **`.panel`** — блоки на внутренних страницах: фон `--bg-card`, рамка, `border-radius`, анимация `fx-panel-border` для панелей (не `ip-box`).
- **Формы**, **кнопки**, **профиль**, **админка** — в отдельных секциях файла с комментариями-разделителями вида `/* ——— ... ——— */`.

### Подвал (текст)

- **`.site-footer`** — типографика, отступы, цвета.
- **`.footer-api-row`** — flex, перенос строки, выравнивание статуса API.
- **`.footer-live-hint`** — скрыта по умолчанию (`display: none`), при показе из JS занимает ширину строки для читаемости.

## Логика (`static/js/app.js`)

### База API

- `API` вычисляется из `window.WC_L_API_BASE` или порта (5500/5501 → хост + порт 8000) или `/api/v1`.
- Токен в `localStorage` под ключом `wc_l_access_token`.

### Навигация

- **`showView(name)`** — выставляет `data-view`, переключает `is-visible` на секциях, подсвечивает кнопки навигации, при необходимости грузит профиль/админку, скроллит в начало страницы.

### Подвал и API

- **`patchFooterLinks`** (IIFE при загрузке) — подставляет абсолютные ссылки на docs/health при отдельном origin и показывает подсказку Live Server.
- **`checkApiStatus`** — запрос к health, обновляет индикатор «●» и текст («API в сети» / ошибка).

### Прочее

В том же файле: регистрация/вход, профиль, админка, анимации монет и звёзд. Заголовок файла в коде описывает Live Server и переопределение API.

## История решений (кратко, для контекста)

Чтобы не повторять откатанные эксперименты:

| Что пробовали | Итог |
|---------------|------|
| Разный отступ футера через `:has(#view-home)` | Заменено на **`data-view`** на `body` для навигации и отладки; отдельные правила футера по экранам не используются в финальной версии. |
| Flex на всём `body` + «прижатый» низ | Откат: ломало согласованность со слоями фона и монет. |
| Рамка вокруг только футера (как карточка) | Откат: визуально мешало, казалось «всплывающим» блоком. |
| **`site-footer-band`** с непрозрачным фоном | **Оставлено** — текст и золотая линия остаются на «сером» низу, не на фото подземелья. |

## Как ориентироваться в `fantasy.css`

1. Сверху вниз: **шрифты**, **`:root`**, **keyframes**, **сбросы**, **`html`/`body`**, **`body::before`**, **`main`/футер**, **монеты**, **шапка**, **кнопки**, **макет страниц**, **панели**, **герой**, **сетка карточек**, **формы**, **подвал**, **профиль**, **медиа‑запросы**.

2. Комментарии на русском помечают **слои** (z-index) и **назначение** блоков — их можно искать по `———`.

## Чек-лист для правок

- Меняете стили — поднимите **`?v=`** у `fantasy.css` в `index.html`.
- Меняете поведение навигации — смотрите **`showView`** и объекты **`views`**.
- Меняете видимость футера на фоне — **`site-footer-band`** и фон **`body`**; не делайте подвал прозрачным без необходимости.
- Проверяйте и **длинную страницу**, и **короткую** (мало контента в `main`), и **узкое окно**.
